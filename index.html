Estructura de datos con Javascript

Hash tables

Es una estructura donde se reserva un espacio de memoria.

Es como un objeto que tiene un key y un value, pero para acceder al valor es necesario indicar un hash.

A los elementos formados por el hash, key y value se lo denomina bucket

Lo bueno de la hashtable es que es opaca, no se puede conoce los valores a menos que se ingrese la dirección (hash) correcto.

Metodos

  • insert: agregar elementos a la hash table
  • search: dado un key buscar su value
  • delete: borrar un bucket
  • keys: devuelve un array con todas las keys de la hash table

Linked List

Es una estructura donde hay:

  • Un head: primer nodo con su valor y apuntando al segundo nodo.
  • nodos que indican su valor y apuntan al siguiente nodo.
  • un tail: es un nodo final con un valor y que apunta a null.

Metodos

  • pretend: Agregar un nodo al inicio que se convierte en el head.
  • append: agregar un nodo al final que se convierte en el tail.
  • lookup/search: buscar un nodo.
  • insert: agregar un nodo en la lista en una posicion.
  • delete: borra un nodo.

Las listas pueden ser SiglyLinkedList o DoublyLinkedList. Las SiglyLinkedList tienen cada nodo apuntando al nodo siguiente, salvo el tail que cuyo siguiente (next) apunta a null. En las DoublyLinkedList cada nodo apunto al nodo siguiente y al anterior, salvo el tail que cuyo siguiente (next) a null y le head que cuyo anterior (prev) a null también.

Stack

Es un stack los datos van llegando uno a uno y se apilan.

Es un LIFO donde los datos se van apilando de tal manera que solo puede accederse y sacarse el último elmento apilado. Es como si fuera una pila de ropa, la ultima prenda colocada arriba es la única que puede accederse.

Métodos

  • pop: elimina el ultimo elemento.
  • push: agrega un elemento al final.
  • peek: toma el elemento del final.

Queue

En un stack FIFO los datos se ponen en fila segun el orden de llegada, el dato que puede accederse y sacarse es el primero en llegar. Es como la fila de banco, la primer persona en llegar es la primera persona en atenderse.

Métodos

  • enqueue: agregar un elemento al final
  • dequeue: eliminar el primer elemento.
  • peek: tomar el primer elemento.
